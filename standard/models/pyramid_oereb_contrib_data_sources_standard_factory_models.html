Document¶

class pyramid_oereb.contrib.data_sources.standard.factory_models.Document(**kwargs)¶

THE DOCUMENT This represents the main document in the whole system.

id¶

The identifier. This is used in the database only and must not be set manually. If you don’t like it - don’t care about.

Type:

int

document_type¶

The document type. It must be “Rechtsvorschrift”, “GesetzlicheGrundlage” or “Hinweis”.

Type:

str

index¶

An index used to sort the documents.

Type:

int

law_status¶

The status switch if the document is legally approved or not.

Type:

str

title¶

The multilingual title or if existing the short title ot his document.

Type:

dict

office_id¶

The foreign key to the office which is in charge for this document.

Type:

str

responsible_office¶

The dedicated relation to the office instance from database.

Type:

pyramid_oereb.standard.models.railways_project_planning_zones.Office

published_from¶

The date from when the document should be available for publishing in extracts. This directly affects the behaviour of extract generation.

Type:

datetime.date

published_until¶

The date until when the document should be available for publishing on extracts. This directly affects the behaviour of extract generation.

Type:

datetime.date

text_at_web¶

A multilingual link which leads to the documents content in the web.

Type:

dict

abbreviation¶

The multilingual shortened version of the documents title.

Type:

dict

official_number¶

The multilingual official number which uniquely identifies this document.

Type:

dict

only_in_municipality¶

The fosnr (=id bfs) of the municipality. If this is None it is assumed the document is related to the whole canton or even the confederation.

Type:

int

file¶

The document itself as a binary representation (PDF). It is string but BaseCode64 encoded.

Type:

str
