Print Proxy¶

The print proxy is a mechanism to provide fully variable pdf print creation process inside of the oereb server with minimal impact on the behavior of it. The idea is to bind an external web accessible print service to the oereb server. The extract is generated by the oereb server the result is sent to the print instance and the created pdf is received and returned to the client. This is a proxy/passtrough mechanism.

Sub themes¶

To enable separate pages per sub theme, enable the option split_sub_themes in the print section of your configuration.

print:
    split_sub_themes: true

You can influence the sort order for each theme separately. Per default, the sub themes are unsorted.

Example for alphabetic sort:

- name: plr119
  sub_themes:
    sorter:
        module: pyramid_oereb.contrib.print_proxy.sub_themes.sorting
        class_name: AlphabeticSort

The list sort provides a fix, non-alphabetic order. A ordered list as parameter defines the order. The sub theme names have to match exactly, otherwise they will be added at the end. Example configuration:

- name: plr119
  sub_themes:
    sorter:
        module: pyramid_oereb.contrib.print_proxy.sub_themes.sorting
        class_name: ListSort
        params:
            list:
                - First sub theme
                - Another sub theme
                - Last sub theme
